## 2. 模块职责划分

### 2.1 serai-collector（串口采集模块）

#### 核心职责
- **串口设备管理**: 初始化、打开、关闭串口连接
- **Modbus-RTU协议实现**: 构建Modbus请求、解析Modbus响应
- **数据采集调度**: 按1秒间隔定时采集温湿度数据
- **数据缓存**: 断网时缓存数据，网络恢复后批量上传
- **TCP客户端**: 维持与rtu-gateway的TCP长连接
- **二进制协议封装**: 将采集数据封装为自定义二进制帧

#### 功能边界
✅ **负责**:
- 串口通信的所有细节（波特率、数据位、校验位等）
- Modbus-RTU协议的CRC校验
- 采集任务的定时调度
- 本地数据缓存和重传机制
- 与rtu-gateway的TCP连接管理

❌ **不负责**:
- 数据的业务逻辑处理
- 数据持久化到数据库
- MQTT消息发布
- Web接口提供

#### 关键类设计

### 2.2 rtu-gateway（RTU网关模块）

#### 核心职责
- **TCP服务端**: 接收多个serai-collector的连接
- **二进制协议解析**: 解析自定义二进制帧，提取传感器数据
- **Modbus数据解析**: 解析Modbus响应中的温湿度值
- **数据验证**: 校验数据完整性和合法性
- **MQTT客户端**: 将数据以JSON格式发布到MQTT Broker
- **连接管理**: 管理多个采集器的连接状态

#### 功能边界
✅ **负责**:
- TCP服务端的启动和连接管理
- 二进制帧的解码和验证
- Modbus数据字段的解析（功能码、寄存器值）
- 数据格式转换（二进制 → JSON）
- MQTT消息的发布
- 连接状态监控和日志记录

❌ **不负责**:
- 串口通信
- 数据采集调度
- 数据持久化
- 业务逻辑处理（报警、统计等）
- Web接口提供

### 2.3 web-server（业务服务模块）

#### 核心职责
- **MQTT订阅**: 订阅rtu-gateway发布的数据主题
- **数据持久化**: 将接收到的数据存储到MySQL数据库
- **HTTP API服务**: 基于Netty实现的HTTP接口服务
- **RTU管理**: RTU设备的注册、配置、状态管理
- **数据查询**: 历史数据查询、统计分析
- **报警处理**: 阈值判断、报警记录、报警推送
- **配置下发**: 通过MQTT向rtu-gateway下发配置指令
- **静态资源服务**: 提供前端页面和静态文件

#### 功能边界
✅ **负责**:
- MQTT消息订阅和处理
- 数据库的增删改查（JDBC/MyBatis）
- HTTP API设计和实现（Netty HTTP Server）
- 业务逻辑处理（报警、统计、权限等）
- 静态资源服务（HTML/CSS/JS）
- 配置管理和下发

❌ **不负责**:
- 串口通信
- Modbus协议处理
- TCP连接管理（与采集器的连接）
- 数据采集调度

#### 技术架构
- **HTTP服务**: Netty HTTP Server（与rtu-gateway保持技术栈一致）
- **数据库访问**: JDBC / MyBatis（轻量级）
- **MQTT客户端**: Eclipse Paho MQTT Client
- **JSON处理**: Gson / Jackson
- **前端**: 原生JavaScript / Vue.js（可选）

## 3. 内部通信接口

### 3.1 serai-collector ↔ 温湿度传感器（Modbus-RTU）

#### 协议类型
Modbus-n
#### 请求格式（读取保持寄存器 0x03）
```
字节索引  | 字段         | 示例值  | 说明
---------|-------------|---------|------------------
0        | 从站地址     | 0x01    | 设备地址
1        | 功能码       | 0x03    | 读保持寄存器
2-3      | 起始地址     | 0x0000  | 寄存器起始地址
4-5      | 寄存器数量   | 0x0002  | 读取2个寄存器
6-7      | CRC校验      | 0xXXXX  | CRC16校验码
```

#### 响应格式
```
字节索引  | 字段         | 示例值  | 说明
---------|-------------|---------|------------------
0        | 从站地址     | 0x01    | 设备地址
1        | 功能码       | 0x03    | 读保持寄存器
2        | 字节数       | 0x04    | 后续数据字节数
3-4      | 温度值       | 0x00FA  | 温度 = 250 / 10 = 25.0°C
5-6      | 湿度值       |  = 500 / 10 = 50.0%
7-8      | CRC校验      | 0xXXXX  | CRC16校验码
```

#### 数据解析规则
- **温度**: `(高字节 << 8 | 低字节) / 10.0`
- **湿度**: `(高字节 << 8 | 低字节) / 10.0`

---

### 3.2 serai-collector ↔ rtu-gateway（自定义二进制协议）

#### 协议类型
TCP长连接 + 自定义二进制帧

#### 数据帧格式
```
字节索引  | 字段         | 长度    | 示例值      | 说明
---------|-------------|---------|------------|------------------
0-1      | 帧头         | 2字节   | 0xAA55     | 固定帧头
2        | 版本号       | 1字节   | 0x01       | 协议版本
3        | 帧类型       | 1字节   | 0x01       | 0x01=数据上报
4-5      | 数据长度     | 2字节   | 0x0020     | 后续数据长度
6-21     | RTU ID      | 16字节  | "RTU001..."| RTU唯一标识   | 时间戳       | 8字节   | 毫秒时间戳  | 采集时间
30-N     | Modbus原始数据| 变长    | 0x01030... | Modbus响应帧
N+1-N+2  | CRC16校验    | 2字节   | 0xXXXX     | 整帧CRC校验
```

#### 帧类型定义
```java
public interface FrameType {
    byte DATA_REPORT = 0x01;      // 数据上报
    byte HEARTBEAT = 0x02;        // 心跳包
    byte CONFIG_ACK = 0x03;       // 配置确认
    byte ERROR_REPORT = 0x04;     // 错误上报
}
```

#### 示例数据
```
AA 55 01 01 00 20
52 54 55 30 30 31 00 00 00 00 00 00 00 00 00 00  // RTU001
00 00 01 8E 3F 2A 5C 80                          // 时间戳
01 03 04 00 FA 01 F4 XX XX                       // Modbus数据
XX XX                                             // CRC
```

## 6. 错误码定义

### 6.1 错误码规范

系统采用分段式错误码设计，便于快速定位问题所属模块：

| 错误码范围 | 模块 | HTTP 状态码映射 |
|------------|------|----------------|
| 200-499 | 通用错误 | 根据具体值映射 |
| 1000-1999 | RTU 管理 | 400 Bad Request |
| 2000-2999 | 数据管理 | 400 Bad Request |
| 3000-3999 | 配置管理 | 400/500 |
| 4000-4999 | 报警管理 | 400/500 |
| 5000-5999 | MQTT 通信 | 500 Internal Server Error |
| 6000-6999 | 数据库 | 500 Internal Server Error |
| 7000-7999 | 权限认证 | 401/403 |

### 6.2 完整错误码列表

#### 通用错误 (200-499)

| 错误码 | HTTP 状态码 | 说明 | 使用场景 |
|--------|-----------|------|----------|
| 200 | 200 OK | 操作成功 | 所有成功操作 |
| 400 | 400 Bad Request | 请求参数错误 | 通用参数错误 |
| 401 | 401 Unauthorized | 未授权访问 | Token 缺失或无效 |
| 403 | 403 Forbidden | 禁止访问 | 权限不足 |
| 404 | 404 Not Found | 资源不存在 | 通用资源不存在 |
| 405 | 405 Method Not Allowed | 方法不允许 | HTTP 方法不支持 |
| 500 | 500 Internal Server Error | 系统内部错误 | 通用系统错误 |

#### RTU 管理错误 (1000-1999)

| 错误码 | HTTP 状态码 | 说明 | 触发场景 |
|--------|-----------|------|----------|
| 1001 | 404 | RTU 不存在 | 查询不存在的 RTU ID |
| 1002 | 400 | RTU 已存在 | 注册重复的 RTU |
| 1003 | 400 | RTU 离线 | 对离线 RTU 下发指令 |
| 1004 | 400 | RTU 已在线 | 重复上线操作 |
| 1005 | 400 | RTU 配置错误 | 配置参数非法 |
| 1006 | 400 | RTU ID 不能为空 | 注册时 rtuId 为空 |

### 6.4 异常处理机制

框架层自动捕获并转换异常为统一 HTTP 响应：

```java
// Controller 中抛出异常
if (!rtuService.exists(rtuId)) {
    throw new NotFoundException(ErrorCode.RTU_NOT_FOUND);
}

// 框架自动处理并返回
// HTTP 404 Not Found
// { "code": 1001, "message": "RTU 不存在" }
```

**HTTP 状态码映射规则：**
- 200 → 200 OK
- 400-499 → 400 Bad Request
- 401 → 401 Unauthorized
- 403 → 403 Forbidden
- 404 → 404 Not Found
- 500+ → 500 Internal Server Error
