## 概述

本文档描述了物联网 RTU 网关系统的 API 接口设计，包括 RTU 设备管理、数据查询、配置下发、报警管理等功能。系统采用 Netty 实现高性能 TCP 通信，通过 MQTT 消息队列进行数据转发，提供 RESTful API 供前端调用。

**系统架构**: serial-collector(边缘采集) ↔ rtu-gateway(协议转换) ↔ MQTT Broker ↔ web-server(业务服务)

**版本**: v2.0.0  
**更新日期**: 2026-03-13

## RTU 管理接口

### 1. RTU 设备注册接口

**接口路径**: `/api/rtu/register`

**请求方法**: POST

**请求头**:
- Content-Type: application/json
- Access-Token: {access_token} (可选)

**请求体**:
```json
{
  "rtuId": "RTU-001",
  "name": "1 号温湿度采集器",
  "location": "机房 A 区",
  "serialPort": "COM3",
  "baudRate": 9600,
  "deviceAddress": 1,
  "samplingInterval": 1
}
```

**字段说明**:
- rtuId: RTU 唯一标识 (必填，全局唯一，如 RTU-001)
- name: RTU 名称 (必填，1-64 字符，支持中文、英文、数字、下划线)
- location: 安装位置 (可选，描述 RTU 部署位置)
- serialPort: 串口号 (必填，如 COM3 或/dev/ttyUSB0)
- baudRate: 波特率 (可选，默认 9600，范围 1200-115200)
- deviceAddress: Modbus 设备地址 (可选，默认 1，范围 1-255)
- samplingInterval: 采集间隔 (可选，单位秒，默认 1 秒)
- secret: 认证密钥 (系统自动生成，创建后不可修改)

**注意事项**:
- rtuId 在全系统中必须唯一
- secret 由系统自动生成并保存，用于 RTU 连接认证
- 注册成功后，RTU 初始状态为 offline

**响应示例**:
```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": 1,
    "rtuId": "RTU-001",
    "name": "1 号温湿度采集器",
    "location": "机房 A 区",
    "status": "offline",
    "serialPort": "COM3",
    "baudRate": 9600,
    "deviceAddress": 1,
    "samplingInterval": 1,
    "secret": "abc123xyz",
    "createTime": "2026-03-13 10:00:00",
    "updateTime": "2026-03-13 10:00:00"
  }
}
```

**错误码**:
- 1002: RTU 已存在
- 1006: RTU ID 不能为空
- 1007: RTU 名称不能为空
- 1009: RTU ID 重复

### 5. RTU 认证验证接口

**接口路径**: `/api/rtu/gateway/verify`

**请求方法**: POST

**请求头**:
- Content-Type: application/json

**请求体**:
```json
{
  "rtuId": "RTU-001",
  "secretHash": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
}
```

**字段说明**:
- rtuId: RTU 唯一标识 (必填)
- secretHash: secret 的 SHA-256 哈希值 (必填)

**功能描述**:
rtu-gateway 接收到 serial-collector 的认证请求后，调用此接口验证 RTU 合法性。web-server 查询数据库验证 rtuId 和 secret 是否匹配，并返回 RTU 状态。

**认证流程**:
1. serial-collector 连接到 rtu-gateway
2. 发送认证帧 (包含 rtuId 和 secret 的哈希)
3. rtu-gateway 调用此接口验证
4. web-server 查询数据库并返回验证结果
5. rtu-gateway 根据结果建立或关闭连接

**响应示例** (认证成功):
```json
{
  "code": 200,
  "message": "认证成功",
  "data": {
    "valid": true,
    "status": "ENABLED",
    "rtuId": "RTU-001"
  }
}
```

**响应示例** (认证失败):
```json
{
  "code": 401,
  "message": "认证失败：RTU 不存在或密钥错误",
  "data": {
    "valid": false
  }
}
```

**错误码**:
- 1001: RTU 不存在
- 7002: Token 无效
- 7007: 用户已禁用

### 6. 配置下发接口

**接口路径**: `/api/rtu/{rtuId}/config`

**请求方法**: POST

**请求头**:
- Content-Type: application/json

**请求体**:
```json
{
  "samplingInterval": 5,
  "temperatureThreshold": {
    "min": 18.0,
    "max": 28.0
  },
  "humidityThreshold": {
    "min": 40.0,
    "max": 60.0
  },
  "alarmEnabled": true
}
```

**字段说明**:
- samplingInterval: 采集间隔 (可选，单位秒，范围 1-60)
- temperatureThreshold: 温度阈值 (可选，包含 min 和 max)
- humidityThreshold: 湿度阈值 (可选，包含 min 和 max)
- alarmEnabled: 是否启用报警 (可选，默认 true)

**功能描述**:
配置下发是异步操作。接口会立即返回成功，实际配置通过以下流程下发:
1. web-server 保存配置到数据库 rtu_config 表
2. 通过 MQTT发布到主题：`iot/rtu/{rtuId}/config`
3. rtu-gateway 接收 JSON 配置，转换为 Modbus 写入命令
4. 封装为二进制帧下发到 serial-collector
5. serial-collector 通过串口写入传感器设备

**响应示例**:
```json
{
  "code": 200,
  "message": "配置更新成功，已下发到 RTU",
  "data": {
    "rtuId": "RTU-001",
    "configStatus": "pending",
    "updateTime": "2026-03-13 11:00:00"
  }
}
```

**错误码**:
- 1001: RTU 不存在
- 1003: RTU 离线，无法下发配置
- 1005: RTU 配置错误
- 3003: 采样间隔无效 (1-60 秒)
- 3005: 温度最小值不能大于最大值
- 3006: 湿度最小值不能大于最大值
- 3007: 配置下发失败

## 数据查询接口

### 1. 实时数据查询接口

**接口路径**: `/api/data/realtime`

**请求方法**: GET

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| rtuId | String | 是 | RTU ID | RTU-001 |

**请求示例**:
```
GET /api/data/realtime?rtuId=RTU-001
```

**响应示例**:
```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "rtuId": "RTU-001",
    "deviceId": "RTU-001",
    "temperature": 25.0,
    "humidity": 50.0,
    "timestamp": "2026-03-13 10:30:00",
    "status": "normal"
  }
}
```

**字段说明**:
- rtuId: RTU 标识
- deviceId: 设备标识 (与 rtuId 相同)
- temperature: 温度值 (单位：℃，范围 -40~85)
- humidity: 湿度值 (单位：%RH，范围 0~100)
- timestamp: 数据采集时间
- status: 数据状态 (normal/warning/error)

**错误码**:
- 1001: RTU 不存在
- 2001: 数据不存在
- 2005: 温度值超出范围 (-40~85°C)
- 2006: 湿度值超出范围 (0~100%)

## 错误码定义

### 通用错误 (200-499)

| 错误码 | HTTP 状态码 | 说明 | 使用场景 |
|--------|-----------|------|----------|
| 200 | 200 OK | 操作成功 | 所有成功操作 |
| 400 | 400 Bad Request | 请求参数错误 | 通用参数错误 |
| 401 | 401 Unauthorized | 未授权访问 | Token 缺失或无效 |
| 403 | 403 Forbidden | 禁止访问 | 权限不足 |
| 404 | 404 Not Found | 资源不存在 | 通用资源不存在 |
| 405 | 405 Method Not Allowed | 方法不允许 | HTTP 方法不支持 |
| 500 | 500 Internal Server Error | 系统内部错误 | 通用系统错误 |

### RTU 管理错误 (1000-1999)

| 错误码 | HTTP 状态码 | 说明 | 触发场景 |
|--------|-----------|------|----------|
| 1001 | 404 | RTU 不存在 | 查询不存在的 RTU ID |
| 1002 | 400 | RTU 已存在 | 注册重复的 RTU |
| 1003 | 400 | RTU 离线 | 对离线 RTU 下发指令或删除操作 |
| 1004 | 400 | RTU 已在线 | 重复上线操作 |
| 1005 | 400 | RTU 配置错误 | 配置参数非法 |
| 1006 | 400 | RTU ID 不能为空 | 注册时 rtuId 为空 |
| 1007 | 400 | RTU 名称不能为空 | 注册时 name 为空 |
| 1008 | 400 | 无效的 RTU 状态 | status 不在枚举范围 |
| 1009 | 400 | RTU ID 重复 | 使用已存在的 ID |

**接口异常示例**:
- `POST /api/rtu/register` - RTU 已存在 → 1002
- `GET /api/rtu/{rtuId}` - RTU 不存在 → 1001
- `DELETE /api/rtu/{rtuId}` - RTU 在线 → 1003

### 数据相关错误 (2000-2999)

| 错误码 | HTTP 状态码 | 说明 | 触发场景 |
|--------|-----------|------|----------|
| 2001 | 404 | 数据不存在 | 查询不到历史数据 |
| 2002 | 400 | 数据格式错误 | JSON 解析失败 |
| 2003 | 400 | 数据超出范围 | 数值超过合理范围 |
| 2004 | 400 | 数据解析失败 | Modbus 解析错误 |
| 2005 | 400 | 温度值超出范围 (-40~85°C) | temperature < -40 或 > 85 |
| 2006 | 400 | 湿度值超出范围 (0~100%) | humidity < 0 或 > 100 |
| 2007 | 400 | 时间戳无效 | 时间戳为负数或格式错误 |

**接口异常示例**:
- `GET /api/data/realtime` - 温度超出范围 → 2005
- `GET /api/data/history` - 湿度超出范围 → 2006
- `GET /api/data/statistics` - 时间戳无效 → 2007

### 配置管理错误 (3000-3999)

| 错误码 | HTTP 状态码 | 说明 | 触发场景 |
|--------|-----------|------|----------|
| 3001 | 404 | 配置不存在 | 查询不存在的配置 |
| 3002 | 400 | 配置格式错误 | 配置 JSON格式错误 |
| 3003 | 400 | 采样间隔无效 (1-60 秒) | interval < 1 或 > 60 |
| 3004 | 400 | 阈值设置错误 | 阈值配置不完整 |
| 3005 | 400 | 温度最小值不能大于最大值 | tempMin >= tempMax |
| 3006 | 400 | 湿度最小值不能大于最大值 | humiMin >= humiMax |
| 3007 | 500 | 配置下发失败 | MQTT 下发配置失败 |

**接口异常示例**:
- `POST /api/rtu/{rtuId}/config` - 采样间隔无效 → 3003
- `POST /api/rtu/{rtuId}/config` - 温度阈值错误 → 3005
- `POST /api/rtu/{rtuId}/config` - 配置下发失败 → 3007

### 报警管理错误 (4000-4999)

| 错误码 | HTTP 状态码 | 说明 | 触发场景 |
|--------|-----------|------|----------|
| 4001 | 404 | 报警记录不存在 | 处理不存在的报警 |
| 4002 | 400 | 报警已处理 | 重复处理报警 |
| 4003 | 400 | 报警类型无效 | alarmType 不在枚举范围 |
| 4004 | 400 | 报警级别无效 | alarmLevel 不在枚举范围 |

**接口异常示例**:

## 安全规范

### 1. 身份认证
- 所有写操作接口建议使用 Access-Token 进行身份验证
- Token 采用 JWT 格式，有效期 24 小时
- Token 过期后需重新登录获取

### 2. 权限控制
- RTU 注册、删除等敏感操作需要管理员权限
- 配置下发需要操作员及以上权限
- 数据查询接口对所有认证用户开放

### 3. 输入校验
- 所有输入参数必须进行合法性校验
- 防止 SQL 注入和 XSS 攻击
- 字符串长度、数值范围都需要验证

### 4. 数据安全
- secret 采用 SHA-256 哈希存储
- 敏感操作需要记录操作日志
- 返回数据不包含敏感信息

### 5. 限流策略
- 单 IP 每分钟最多 1000 次请求
- 配置下发接口每秒最多 10 次
- 超过限制返回 429 Too Many Requests
