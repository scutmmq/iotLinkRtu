## 运行测试

### 方式1：运行所有测试

```bash
mvn test
```

### 方式2：运行特定测试类

```bash
# 运行 Modbus 协议测试
mvn test -Dtest=ModbusProtocolTest

# 运行二进制协议测试
mvn test -Dtest=BinaryProtocolTest

# 运行集成测试
mvn test -Dtest=DataCollectionFlowTest
```

1. 打开测试类
2. 右键点击类名或方法名
3. 选择 "Run 'TestName'" 或 "Run 'methodName()'"

```bash
mvn test -Dtest=AllTests
```

## 测试覆盖的关键场景

### 数据解析场景

1. **正常温湿度**：26.5℃, 58.4%RH
2. **负温度**：-10.1℃, 65.8%RH
3. **零值**：0℃, 0%RH
4. **极值**：100℃, 100%RH / -40℃
5. **无效数据**：长度错误、CRC错误、功能码错误

### 协议场景

1. **认证流程**：rtuId + SHA-256 加密 secret
2. **数据上报**：Modbus 响应 → 二进制帧
3. **配置下发**：二进制帧 → Modbus 命令
4. **心跳机制**：定时心跳帧

### 边界条件

1. **超长 rtuId**：自动截断为 16 字节
2. **短 rtuId**：自动补 0
3. **空数据**：正确处理空 Modbus 数据
4. **大数据**：支持 64 字节以上数据

## 测试数据说明

### Modbus 寄存器映射

| 寄存器地址 | 功能 | 数据格式 |
|-----------|------|---------|
| 0x0000 | 湿度 | 扩大10倍（58.4% → 584） |
| 0x0001 | 温度 | 扩大10倍，负数补码（-10.1℃ → -101 → 0xFF9B） |
| 0x0050 | 温度校准值 | 扩大10倍 |
| 0x0051 | 湿度校准值 | 扩大10倍 |
| 0x07D0 | 设备地址 | 1-254 |
| 0x07D1 | 波特率 | 0=2400, 2=9600, 等 |

### 二进制帧格式

```
+--------+--------+----------+----------+--------+
| 帧头   | 帧类型 | 数据长度 | 数据体   | 校验和 |
| 2字节  | 1字节  | 2字节    | N字节    | 1字节  |
+--------+--------+----------+----------+--------+

帧头：0xAA55
帧类型：0x01=认证, 0x03=数据, 0x04=命令, 0x05=心跳
数据长度：大端序
校验和：异或校验
```

## 故障排查

### 测试失败常见原因

1. **配置文件缺失**
   - 确保 `src/main/resources/config.properties` 存在
   - 检查配置项是否完整

2. **CRC 校验失败**
   - 检查 Modbus 数据是否正确
   - 验证 CRC16 计算算法

3. **数据解析错误**
   - 确认温湿度值的扩大倍数（10倍）
   - 检查负温度的补码处理

4. **帧结构错误**
   - 验证帧头 0xAA55
   - 检查数据长度字段（大端序）
   - 确认校验和计算

## 扩展测试

如需添加新测试：

1. 在对应包下创建测试类
2. 使用 `@Test` 注解标记测试方法
3. 使用 JUnit 5 断言进行验证
4. 添加清晰的测试输出日志

示例：

```java
@Test
public void testNewFeature() {
    // 准备测试数据
    byte[] data = ...;

    // 执行测试
    Result result = someMethod(data);

    // 验证结果
    assertNotNull(result);
    assertEquals(expected, result.getValue());

    System.out.println("✓ 新功能测试通过");
}
```
